# Internationalization (i18n)

## Configuration

```php
// config/i18n.php
return [
    'default' => env('I18N_LOCALE', 'en'),
    'fallback' => env('I18N_FALLBACK', 'en'),
    'prefixes' => [
        'en' => '/',
        'fr' => '/fr',
    ],
];
```

- **default** — the active locale when no prefix is detected
- **fallback** — used when a translation key is missing in the active locale
- **prefixes** — URL prefix per locale. The default locale uses `/`

## Translation Files

JSON files in `locales/`:

```json
// locales/en.json
{
    "welcome": "Welcome to Colibri",
    "cart": {
        "empty": "Your cart is empty",
        "items": "{count} item(s) in your cart"
    }
}
```

```json
// locales/fr.json
{
    "welcome": "Bienvenue sur Colibri",
    "cart": {
        "empty": "Votre panier est vide",
        "items": "{count} article(s) dans votre panier"
    }
}
```

## Translating

### PHP

```php
t('welcome');                            // "Welcome to Colibri"
t('cart.empty');                          // dot notation
t('cart.items', ['count' => 3]);         // "3 item(s) in your cart"
```

### Latte

Three equivalent syntaxes:

```latte
{t 'welcome'}                           {* tag (recommended) *}
{t 'cart.items', count: 3}              {* tag with parameters *}
{'welcome'|t}                           {* filter *}
{t('welcome')}                          {* function *}
```

### Fallback

If a key is missing in the active locale, Colibri looks in the fallback locale. If still not found, the key itself is returned.

## Locale Detection

The router automatically detects the locale from the URL prefix:

- `/about` → locale `en` (default), route `/about`
- `/fr/about` → locale `fr`, route `/about`

The same route files serve all locales — no duplication.

## Locale API

```php
use Colibri\View\Lang;

Lang::locale();            // 'en' — active locale
Lang::setLocale('fr');     // change locale (stored in session)
Lang::locales();           // ['en' => '/', 'fr' => '/fr']
Lang::isLocale('en');      // true
```

In Latte:

```latte
{locale()}                 {* 'en' *}
{locales()}                {* array of locale => prefix *}
```

## URL Helper

Build locale-aware URLs:

```php
url(path: 'about');                       // /about (current locale)
url(path: 'about', locale: 'fr');         // /fr/about
url(locale: 'fr');                        // current page in French
url(path: 'search', query: ['q' => 'test']);  // /search?q=test
url(path: 'about', anchor: 'team');       // /about#team
url();                                    // current URL
```

In Latte:

```latte
<a href="{url(path: 'about')}">About</a>
<a href="{url(locale: 'fr')}">Version française</a>
```

## Language Switcher

```latte
{foreach locales() as $loc => $base}
    <a href="{url(locale: $loc)}"
       style="{if $loc === locale()}font-weight:bold{/if}">
        {$loc|upper}
    </a>{sep} | {/sep}
{/foreach}
```

## Site Title

The `site_title()` function formats the page title with the site name:

```latte
<title>{site_title($page->title)}</title>
```

Configured in `config/app.php`:

```php
'title_format' => ':title | :site',  // "About | MyApp"
```

When the page title equals the site name, it returns just the site name (no duplication).
